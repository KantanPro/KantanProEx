# Coding Rules for This Project

## General Rules
- 既存機能を壊さないこと（デグレ禁止）。
- 不要なリファクタリングはしない。
- 指定された箇所のみ修正する。他は絶対に変更しない。
- 関数名・変数名・クラス名は変更しない。
- 出力は可能な限り **差分（diff形式）** にする。

## Testing
- 修正後も既存テストが通ることを前提にする。
- 新しい機能を追加する場合は必要に応じてテストコードも追加する。

## Documentation
- 修正点にはコメントを残す。
- AIが出力する場合、変更点と理由を必ず説明する。
